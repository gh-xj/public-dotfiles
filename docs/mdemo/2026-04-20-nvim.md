# Neovim learning memo

This note tracks the current user-facing Neovim workflows in
`public-dotfiles/.config/nvim`.

Update it when Neovim keymaps, commands, discovery flows, LSP behavior, or
other operator-facing workflows change.

## Basics

- Leader key is `,`.
- LSP keymaps are buffer-local and appear only after an LSP server attaches.
- Your config uses Telescope for a lot of discovery flows, but symbol
  references currently use built-in LSP quickfix behavior.

## Find symbol references

- Put the cursor on a symbol and press `gA`.
- That mapping calls `vim.lsp.buf.references()`.
- In the current Neovim version in use (`0.12.x`), references go into the
  quickfix list and Neovim opens the quickfix window.
- Quickfix basics:
  - `:cnext` jumps to the next reference
  - `:cprev` jumps to the previous reference
  - `:copen` reopens the quickfix list
  - `:cclose` closes the quickfix list

## Find symbols

- `gs` shows document symbols for the current file through Telescope.
- `gS` shows workspace symbols through Telescope.
- `,o` toggles the Aerial outline panel.

## Discover commands and mappings

- `,c` opens `Telescope commands`.
- `,tk` opens `Telescope keymaps`.
- `,th` opens `Telescope help_tags`.
- `:command` lists custom user commands.
- `:map` or `:nmap` lists mappings.
- `:verbose nmap gA` shows where the `gA` mapping came from.

## Useful custom commands

- `:CloseBufferNoLayout`
- `:LazyGit` opens lazygit in a floating terminal. Press `,gg` to launch it,
  and press `<Esc><Esc>` inside the lazygit terminal to close the float without
  sending a single escape to lazygit.
- `:Format`
- `:NvimMachineReload`
- `:NvimThemeSync`

## Lazygit workflow

- `,gg` runs `:LazyGit`.
- The lazygit float starts in the Git root for the current buffer when one can
  be found; otherwise it falls back to Neovim's current working directory.
- The lazygit float passes the current Neovim server address through `$NVIM`.
- Lazygit uses its `nvim-remote` editor preset, so pressing `e` in lazygit
  opens the selected file in the parent Neovim instance instead of nesting a
  second Neovim process inside the terminal.
- Lazygit uses `delta` as the first diff pager with editor hyperlinks. Cycle
  configured pagers from lazygit with `|`; the second pager is `difftastic`.
- In lazygit, `X` opens the custom GitHub actions menu. It can open the repo,
  view or create the current branch PR, compare the current branch with the
  default main branch, copy branch names, and open selected commits/files.

## Benchmark markdown open performance

- Compare your full config against baseline:
  `hyperfine --warmup 3 --runs 10 'nvim /path/to/file.md +qa!' 'nvim --clean /path/to/file.md +qa!'`
- Capture startup detail for one markdown file:
  `nvim --startuptime /tmp/nvim-markdown-startup.log /path/to/file.md +qa!`
- For this config, the markdown hot path is:
  - `BufReadPost`: Treesitter setup from `.config/nvim/lua/plugins/treesitter.lua`
  - `FileType markdown`: markdown plugins from `.config/nvim/lua/plugins/markdown.lua`
  - markdown-local buffer options from `.config/nvim/lua/config/autocmds.lua`
- Treesitter-based folding is no longer global. It is enabled buffer-locally for
  supported filetypes, so empty startup and unsupported buffers do not pay
  Treesitter fold setup by default.
- Large markdown files now skip the expensive markdown UI plugins entirely.
  Current threshold: larger than `512 KB` or more than `5000` lines.
- Large markdown files also skip soft-wrap layout features and markdown-local
  folding/UI settings. That means `wrap`, `linebreak`, `breakindent`,
  conceal-heavy rendering, and fold columns stay off for those big buffers.
- Large markdown files also skip markdown LSP bootstrap and repo-aware Git signs
  auto-attach on open, so they avoid `marksman`, `cmp-nvim-lsp`, and
  `gitsigns.nvim` unless you opt back in.
- For smaller markdown files, `render-markdown.nvim` and `mkdnflow.nvim`
  still auto-load on `FileType markdown`.
- For large markdown files, you can opt back in manually:
  - `:RenderMarkdown enable`
  - `:Mkdnflow start`
  - `:Gitsigns attach`
- Theme syncing now runs silently on `VimEnter` and `VimResume`. Use
  `:NvimThemeSync` if you want to force a manual resync immediately.
- Built-in spell checking is no longer enabled by default for markdown buffers,
  so typo underlines in markdown should stay off unless you turn `:setlocal spell`
  on manually.

## Notes on command discovery

- `which-key.nvim` is installed, so leader-based key discovery is available in
  the setup, even though there is no custom `which-key` config yet.
- For raw inspection, built-in `:command`, `:map`, and `:verbose map ...` are
  still the most direct way to see what is active.

## Source of truth

- Leader key: `.config/nvim/init.lua`
- Telescope and general mappings: `.config/nvim/lua/config/keymaps.lua`
- LSP mappings: `.config/nvim/lua/plugins/lsp.lua`
- UI helpers like `which-key.nvim`: `.config/nvim/lua/plugins/ui.lua`
