# Architecture Attack Report — repo root

## Executive summary

Top findings ranked by `severity × confidence × lens-coincidence`. One line each.

1. **Fragmented config delivery** — the repo ships configs through four different ownership modes, so adding one more config starts with delivery semantics instead of intent. → Collapse delivery into three explicit classes.
2. **Terminal contract split across apps** — Ghostty, tmux, Karabiner, and the verifier all encode the same interaction model separately. → Introduce one keybinding source of truth and generate the rest.
3. **Shell init became a runtime** — `.zshrc` now mixes user ergonomics, caching, compatibility fallbacks, and SSH host indexing. → Split it into a thin orchestrator plus small sourced modules.
4. **macOS baseline uses parallel policy systems** — nix-darwin defaults and TSV-driven live/default ledgers both own adjacent behavior. → Write one placement rule for which layer owns what.
5. **Bootstrap entrypoints duplicate pinning logic** — flake inputs, Taskfile, and generated bootstrap flakes all participate in release selection. → Make the flake the only pin source and reduce generated bootstrap policy.

#### Fragmented config delivery

- **Principle tags:** DRY, KISS, SRP
- **Severity:** high · **Confidence:** 93 · **Blast radius:** cross-cutting
- **Why it hurts:** New config work starts with “which transport mechanism?” rather than “what should this config mean?”.
- **Minimal fix:** Document and enforce exactly three delivery classes: immutable link, mutable seed, generated shim.
